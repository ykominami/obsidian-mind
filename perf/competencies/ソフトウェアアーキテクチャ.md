# ソフトウェアアーキテクチャ

## Definition

システムの構造・インターフェース・コンポーネント分割を設計し、技術的な意思決定を文書化する能力。TECS文脈ではCDL（Component Description Language）によるコンポーネント設計、tecsgen/TECSCDEのアーキテクチャ判断を含む。

## Proficiency Levels

| Level | Description |
|-------|-------------|
| beginner | 既存アーキテクチャを理解し、定められた範囲で実装できる |
| intermediate | サブシステムの設計判断を行い、トレードオフを文書化できる |
| advanced | システム全体のアーキテクチャを設計し、他者の設計をレビューできる |
| expert | 組織の技術方向性を定め、アーキテクチャ標準を策定できる |
