第2章　Claude Codeの基本構造と特徴

「思考の設計図」を扱うことを重視
仕様をどこで区切るか
どの情報を前提条件として固定するか
どの部分に人間の判断余地を残すか

軽量モデル向けの局所的でシンプルなタスク
・既存関数の一部変更
・仕様が明確な範囲でのコード追加
・単一ファイル内で完結する処理の生成
・エラー個所が明白なバグ修正

高性能モデルが真価を発揮する、構造理解が欠かせないタスク
・複数ファイルにまたがる依存関係の整理
・既存コード全体を踏まえたアーキテクチャの見直し
・あいまいな仕様からの構成案・設計案の作成
・「なぜこの実装になっているのか」の背景説明

任せる仕事の性質からモデルを選ぶ

Claude Codeが得意とする領域
・仕様文書から構造を起こす
・既存コードの意図を読み解く
・リファクタリングの”理由”を説明する
・コードとドキュメントの整合性を整える

「このクラスにはどんな責務を持たせているのか」
「なぜここで例外処理を分ける設計にしているのか」

Claude Codeの真価
・人間の思考を言葉にし、それを確認・調整するためのAI

Claude Codeの不得意な点
・UIデザインや細かな見た目の調整
・画像・動画などビジュアル主体の処理
・極めて特殊で最新動向の速い専門領域（最新フレームワークの細部実装など)

3つの前提
1.指示を設計レベルで出すこと
「何を書いてほいいか」だけでなく「なぜそうするのか」「どんな前提があるのか」といった背景を含める
2.出力を必ず問い直す
生成された案に対して「この設計の弱点はどこか」「ほかにとりえる選択肢はあるか」と再質問すると、より深い検討が引き出せる
3.改善前提で使う。

今すぐできること
既存のコードを１つ選び、「この設計意図を説明してください」と依頼する。

プロンプト設計
プロンプトの原則
1. 前提・目的・制約を必ず書く
2. 段階的に依頼する（大規模タスクを分割）
3. アウトプット形式を指定する（例：関数単位・クラス単位）
4. 評価基準を提示する（品質・速度・拡張性など）
5. 改善を前提にフィードバックを繰り返す

よくあるトラブル
・文脈不足による誤解
->前提を３倍書くと改善される

要求が抽象的すぎる
->抽象度を下げて「例」を書くと劇的に精度が上がる

タスクが大きすぎる
->モジュールごとに分けて依頼する

ネットワークや認証の問題
->企業ネットワークでは事前に接続テストを行う習慣をつける。

長文出力で処理が途切れる
->「段階出力」や「箇条書きベース」を指示して会費

第５章　Claude Codeでできること

時間のかかる工程
・仕様を正しく理解する
・設計上の判断が妥当か検討する
・既存コードの意図を読み取る
・修正による景況範囲を洗い出す
・複数の方針を比較し、最適解を選ぶ

Claude Codeが役割
1. 自然言語ベースでのコード生成
2. 定型処理・テンプレート構造の構築
3. 既存コードの構造把握とリファクタリング支援
4. バグや不具合の原因整理と修正方針の提示
5. コードを起点としたドキュメント派生生成
生成->解釈->評価->改善という流れを、一貫した文脈ななかd繰り返せる点が強み

どの前提を持たせて生成させるか

以下のような前提を含める
・関数の役割と責務
・呼び出される文脈
・入出力の条件や製薬
・エラー発生時の扱い方

さらに
・なぜこの構造になっているか
・ほかの設計案との比較
・採用しなかった理由
といた判断の背景まで説明させることができる

コードと同時に思考の履歴を残せる点は、後工程の理解・レビュー・引継ぎにおいて重要

第９章よくあるトラブルとその対処法
応答が途中で止まる原因とその対処
よくある原因
・長大な仕様書や要件を一度に入力した
・複雑な関数やクラス構造をまとめて解析した
・モジュール間の依存関係が多く、モデルが処理しきれない状態になった。
対応策
1.タスクを分割する
例：モジュールごと、関数単位、処理フロー単位で順番に生成来する
2.外部ファイルやリンクで情報を渡す
長文や大量コードは、外部ファイルを参照させて、AIに扱わせることで、モデルの負荷を軽減できます。
3.モデルの使い分け
・Sonetモデル：軽量タスク向け
・Opusモデル：大規模解析や複雑構造のレビュー向け

ファイル・コード量制限への対応
・ファイル分割：大きなコードは複数の小さなファイルに分割してアップロード
・モジュール単位生成：関数単位、クラス単位で生成
・段階的生成：主要コード生成後にテストコードや補助関数を依頼

セキュリティ上の注意点（社内データ利用時）
・機密情報の入力回避
社外秘や個人情報はダミーデータを使い、AIに処理
・アクセス権管理
APIキーやWeb UIへのアクセスは権限を限定し、操作履歴やログ管理も推奨
・生成コードのレビュー
AIが生成したコードは必ず人間がレビューし、外部接続や、権限操作を含む処理は特に注意する。
・社内教育
AI活用のルールや注意点を社内で共有し、安全に使える環境を整備する。
