# Karpathy — LLMコーディングガイドライン

→ **コーディング前に考える** 前提を述べる。不確かなときは聞く。決して推測しない。

→ **シンプルさを優先** 問題を解決する最小限のコードを書く。誰も求めていない抽象化はしない。

→ **外科手術的な変更** リクエストに関係ないコードには触れない。変更したすべての行が、リクエストされた内容に遡れるようにする。

→ **目標指向の実行** 曖昧な指示を検証可能な成功基準に変える。1行も書く前に。
