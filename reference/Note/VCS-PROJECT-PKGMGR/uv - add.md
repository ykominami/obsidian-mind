プロジェクトに依存関係を追加します。

依存関係はプロジェクトの `pyproject.toml` ファイルに追加されます。

指定された依存関係が既に存在する場合は、新しいバージョン指定子に更新されます。
ただし、既存の指定子と異なるマーカーが含まれている場合は、その依存関係の別のエントリが追加されます。

ロックファイルとプロジェクト環境は、追加された依存関係を反映するように更新されます。ロックファイルの更新をスキップするには、`--frozen` を使用します。環境の更新をスキップするには、`--no-sync` を使用します。

要求された依存関係が見つからない場合、uv はエラーで終了します。
ただし、`--frozen` フラグが指定されている場合は、uv は依存関係の存在やプロジェクトとの互換性を確認せずに、そのまま依存関係を追加します。

uv は、現在のディレクトリまたは親ディレクトリでプロジェクトを検索します。プロジェクトが見つからない場合、uv はエラーで終了します。

使用方法: uv add [オプション] <パッケージ|--requirements <要件>

引数:
[パッケージ]...
PEP 508 要件として追加するパッケージ (例: `ruff==0.5.0`)

オプション:
-r, --requirements <要件>
指定されたファイルにリストされているパッケージを追加します。

以下の形式がサポートされています: `requirements.txt`、インラインメタデータを含む `.py` ファイル、`pylock.toml`、`pyproject.toml`、`setup.py`、および
`setup.cfg`。

-c, --constraints <制約>
指定された要件ファイルを使用してバージョンを制約します。

制約ファイルは `requirements.txt` に似たファイルで、インストールされる要件の _バージョン_ のみを制御します。制約はプロジェクトの `pyproject.toml` ファイルには追加されませんが、依存関係の解決時には考慮されます。

これは pip の `--constraint` オプションと同等です。

[env: UV_CONSTRAINT=]

-m, --marker <MARKER>
追加されたすべてのパッケージにこのマーカーを適用します。

--dev
要件を開発用依存関係グループに追加します。

このオプションは `--group dev` のエイリアスです。

[env: UV_DEV=]

--optional <OPTIONAL>
指定された追加機能に対するパッケージのオプション依存関係に要件を追加します。

その後、`--extra` フラグを使用してプロジェクトをインストールすると、グループが有効になります。

この要件に対してオプションの追加機能を有効にするには、`--extra` を参照してください。

--group <GROUP>
指定された依存関係グループに要件を追加します。

これらの要件は、プロジェクトの公開メタデータには含まれません。

--editable
要件を編集可能として追加します。

--raw
提供された依存関係を追加します。

デフォルトでは、UV は `tool.uv.sources` セクションを使用して、Git、ローカル、編集可能、および直接 URL の要件のソース情報を記録します。
`--raw` が指定された場合、UV は `tool.uv.sources` ではなく `project.dependencies` にソース要件を追加します。

さらに、デフォルトでは、UV は依存関係に境界を追加します（例：`foo>=1.0.0`）。`--raw` が指定された場合、UV は境界なしで依存関係を追加します。

--bounds <BOUNDS>
依存関係を追加する際に使用するバージョン指定子の種類。

プロジェクトに依存関係を追加する際、制約または URL が指定されていない場合は、パッケージの最新の互換性バージョンに基づいて制約が追加されます。デフォルトでは、下限制約（例：`>=1.2.3`）が使用されます。

`--frozen` が指定された場合、解決は行われず、依存関係は常に制約なしで追加されます。

このオプションはプレビュー段階であり、将来のリリースで変更される可能性があります。

指定可能な値:
- lower: 下限値のみ。例: `>=1.2.3`
- major: 同じメジャーバージョンを許可。セマンティックバージョンキャレットと同様。例:
`>=1.2.3, <2.0.0`
- minor: 同じマイナーバージョンを許可。セマンティックバージョンチルダと同様。例:
`>=1.2.3, <1.3.0`
- exact: 正確なバージョンを固定。例: `==1.2.3`

--rev <REV>
Git から依存関係を追加する際に使用するコミット

--tag <TAG>
Git から依存関係を追加する際に使用するタグ

--branch <BRANCH>
Git から依存関係を追加する際に使用するブランチ

--lfs
Git から依存関係を追加する際に Git LFS を使用するかどうか

--extra <EXTRA>
依存関係に対して有効にする追加オプション。

複数回指定できます。

この依存関係をオプションのエクストラに追加するには、`--optional` を参照してください。

--no-sync
仮想環境の同期を回避します。

[env: UV_NO_SYNC=]

--locked
`uv.lock` が変更されないことをアサートします。

ロックファイルが最新である必要があります。
