  シグニチャとは何か（tecs-0 wiki）

  シグニチャ（signature）は、関数頭部（function head）の集合であり、TECS コンポーネント間のインタフェース規格です。
  呼び口と受け口に対応付けられ、それらの型整合性を保証する指標となります。

  ただし、構文的整合性のみ保証します。
  意味的整合性（「この関数を呼んだときに何が起きるか」）は各シグニチャの仕様書が担います。

  CDL での定義構文

  signature signature_name {
      type_specifier function_name(param_list);
      [oneway] type_specifier async_func(param_list);
  };

  - oneway 指定で非同期呼出し可能を示せます。

  呼び口・受け口への対応付け

  call  sSignature cPortName;    // 呼び口（機能を要求する口）
  entry sSignature ePortName;    // 受け口（機能を提供する口）
  call  sSignature cArray[];     // 呼び口配列
  entry sSignature eArray[N];    // 受け口配列

  パラメータ指定子

  各関数引数には方向指定子が必須です。

  ┌────────────────────┬────────────────────────┬─────────────────────────────────────────┐
  │       指定子       │          方向          │              パラメータ型               │
  ├────────────────────┼────────────────────────┼─────────────────────────────────────────┤
  │ in                 │ 入力                   │ 非ポインタ or 単一ポインタ（const必須） │
  ├────────────────────┼────────────────────────┼─────────────────────────────────────────┤
  │ out                │ 出力                   │ 単一ポインタ（呼び元がメモリ準備）      │
  ├────────────────────┼────────────────────────┼─────────────────────────────────────────┤
  │ inout              │ 双方向                 │ out と同様                              │
  ├────────────────────┼────────────────────────┼─────────────────────────────────────────┤
  │ send(allocator)    │ 入力（アロケータ渡し） │ 単一ポインタ。呼出し先が dealloc 責任   │
  ├────────────────────┼────────────────────────┼─────────────────────────────────────────┤
  │ receive(allocator) │ 出力（アロケータ渡し） │ 二重ポインタ。呼出し先が alloc 責任     │
  └────────────────────┴────────────────────────┴─────────────────────────────────────────┘

  size_is、count_is、string でポインタ型を修飾できます。send/receive を使う場合、アロケータのシグニチャは alloc/dealloc
  関数を持つ必要があります。

  命名規則

  ┌──────────────────────┬────────────────────┬───────────────────┐
  │     コンテキスト     │      接頭文字      │        例         │
  ├──────────────────────┼────────────────────┼───────────────────┤
  │ タスクコンテキスト   │ s（2文字目大文字） │ sTask, sSemaphore │
  ├──────────────────────┼────────────────────┼───────────────────┤
  │ 非タスクコンテキスト │ si                 │ siTask            │
  └──────────────────────┴────────────────────┴───────────────────┘

  シグニチャの名前空間

  CDL には6種類の名前空間があり、そのひとつが「シグニチャ関数名前空間」（各シグニチャ内で仕切られる）です。
