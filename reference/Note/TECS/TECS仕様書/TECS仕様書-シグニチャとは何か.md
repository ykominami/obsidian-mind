● シグニチャとは何か — TECS仕様書（tecs-0 wiki 版）

  シグニチャ（signature）
  は関数頭部（関数ヘッダ）の集合です。呼び口・受け口に対応付けられ、それらの構文的整合性を保証します。

  CDL でのシグニチャ定義

  signature sMyInterface {
      ER   func1( [in]  int32_t  inval,
                  [out] int32_t *outval );
      void func2( [in]  int32_t  param );
      [oneway] void asyncFunc( [in] int32_t param );   // 非同期
  };

  慣習としてシグニチャ名の先頭は s（例: sTask, sSemaphore）。

  BNF

  signature signature_name '{' function_head_list '}' ';'
  function_head: type_specifier_qualifier_list declarator ';'
               | '[' 'oneway' ']' type_specifier_qualifier_list declarator ';'

  シグニチャの特性

  ┌──────────────────────────┬────────────────────────────────────────────────────────────┐
  │           特性           │                            内容                            │
  ├──────────────────────────┼────────────────────────────────────────────────────────────┤
  │ 構文的整合性のみ保証     │ シグニチャは型チェック（関数シグネチャの一致）を保証する   │
  ├──────────────────────────┼────────────────────────────────────────────────────────────┤
  │ 意味的整合性は保証しない │ 「正しく使われるか」は各シグニチャの仕様書が担う           │
  ├──────────────────────────┼────────────────────────────────────────────────────────────┤
  │ oneway                   │ 非同期呼出し可能であることを示す指定子。戻り値は void のみ │
  └──────────────────────────┴────────────────────────────────────────────────────────────┘

  パラメータ方向指定子

  シグニチャ内の各関数引数には方向指定子が必須：

  ┌────────────────────┬────────────────────────┬────────────────────────────────────┐
  │       指定子       │          方向          │            パラメータ型            │
  ├────────────────────┼────────────────────────┼────────────────────────────────────┤
  │ in                 │ 入力                   │ 非ポインタ または const ポインタ   │
  ├────────────────────┼────────────────────────┼────────────────────────────────────┤
  │ out                │ 出力                   │ 単一ポインタ（呼び元がメモリ準備） │
  ├────────────────────┼────────────────────────┼────────────────────────────────────┤
  │ inout              │ 双方向                 │ out と同様                         │
  ├────────────────────┼────────────────────────┼────────────────────────────────────┤
  │ send(allocator)    │ 入力（アロケータ渡し） │ 呼出し先が dealloc 責任            │
  ├────────────────────┼────────────────────────┼────────────────────────────────────┤
  │ receive(allocator) │ 出力（アロケータ渡し） │ 呼出し先が alloc 責任              │
  └────────────────────┴────────────────────────┴────────────────────────────────────┘

  配列ポインタには size_is(expr) / count_is(expr) / string を組み合わせて修飾します。

  呼び口・受け口との関係

  celltype tMyType {
      call  sMyInterface cPort;    // 呼び口：sMyInterface シグニチャを要求
      entry sMyInterface ePort;    // 受け口：sMyInterface シグニチャを提供
  };

  シグニチャが一致することで呼び口と受け口の結合が許可されます。

  callback 指定子（コンポーネントモデル側）

  シグニチャ自体には callback 指定子を付けることができ、コールバック用途であることを示します（受け口 →
  呼び口の逆結合を可能にする）。
