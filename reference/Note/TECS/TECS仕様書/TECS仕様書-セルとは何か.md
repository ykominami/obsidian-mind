● セルとは何か（tecs-0 wiki 版 — 仕様書原文ベース）

  セル（cell） はコンポーネントインスタンスです。セルタイプのインスタンスとして生成され、以下の4要素を持ちます：

  ┌────────┬──────┬────────────────────────────────────────────────────────┐
  │  要素  │ 配置 │                          説明                          │
  ├────────┼──────┼────────────────────────────────────────────────────────┤
  │ 受け口 │ —    │ 機能を提供する口。シグニチャに対応付けられる           │
  ├────────┼──────┼────────────────────────────────────────────────────────┤
  │ 呼び口 │ —    │ 他のセルの機能を利用する口。シグニチャに対応付けられる │
  ├────────┼──────┼────────────────────────────────────────────────────────┤
  │ 属性   │ ROM  │ 読出し専用パラメータ。セル記述で初期値を上書き可能     │
  ├────────┼──────┼────────────────────────────────────────────────────────┤
  │ 変数   │ RAM  │ 読み書き可能な内部状態                                 │
  └────────┴──────┴────────────────────────────────────────────────────────┘

  呼び口と受け口の特性

  call  sSignature cPortName;    // 呼び口：機能を要求
  entry sSignature ePortName;    // 受け口：機能を提供
  call  sSignature cArray[];     // 呼び口配列（サイズ省略可）
  entry sSignature eArray[N];    // 受け口配列

  - 呼び口 は必ず受け口と結合しなければならない
  - 合流（多対一）: 複数の呼び口が1つの受け口に結合 — 可
  - 分流（一対多）: 1つの呼び口が複数受け口に結合 — 不可
  - コールバック: 2つのセルが相互に呼び口・受け口を持って接続し合う補助的な結合

  セルの結合（組上げ記述）

  cell tCellType CellName {
      cCallPort = TargetCell.eEntryPort;   // 呼び口の結合
      attr = 42;                           // 属性の初期値
  };

  [through(plugin, arg)] 指定子でスルーセルを挿入できます（RPC チャネル生成等）。

  複合セルタイプ内のセル

  複数のセルを組み合わせた複合セルタイプ内では、内部セルのプロトタイプ宣言により相互参照が可能です：

  composite tCompositeName {
      entry sSig eOuter;
      cell tInner inner1 { ... };   // 内部セル
      eOuter = inner1.eInner;       // 外部結合
  };

  リージョンとセルの配置

  セルはリージョン（region）に配置でき、リージョン間の結合が制限されます。デフォルトはリージョン間結合禁止。in_through /
  out_through 指定でスルーセル（RPC 等）を挿入することで結合を許可できます。

  RPC（リモート呼出し）でのセル間通信

  セル間の結合を RPC チャネルコンポーネントに置き換えると分散処理が可能：

  ┌────────────────────────────────────┬────────────────────────────────────┐
  │              接続分類              │                説明                │
  ├────────────────────────────────────┼────────────────────────────────────┤
  │ トランスペアレント（直接）         │ 通常の関数呼出し。同一アドレス空間 │
  ├────────────────────────────────────┼────────────────────────────────────┤
  │ トランスペアレント（アウトタスク） │ 同一CPU・異なるタスク              │
  ├────────────────────────────────────┼────────────────────────────────────┤
  │ オペイク（クローズドネットワーク） │ 静的接続・ポインタ値コピー渡し     │
  ├────────────────────────────────────┼────────────────────────────────────┤
  │ オペイク（オープンネットワーク）   │ 動的接続・認証なし                 │
  └────────────────────────────────────┴────────────────────────────────────┘
