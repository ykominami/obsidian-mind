● 受け口とは何か — TECS仕様書（tecs-0 wiki 版）

  受け口（entry port） はセルが外部に提供する機能の口です。シグニチャに対応付けられ、他のセルの呼び口（call
  port）と結合されます。

  CDL での受け口宣言

  entry sSignature ePortName;    // 単一受け口
  entry sSignature eArray[N];    // 受け口配列

  慣習として受け口名の先頭は e（例: eTask, eKernel）。

  受け口の特性

  ┌──────────────────┬──────────────────────────────────────────────────────────────────────┐
  │       特性       │                                 内容                                 │
  ├──────────────────┼──────────────────────────────────────────────────────────────────────┤
  │ 機能提供         │ 呼び口からの関数呼出しを受け取り、実装された機能を提供する           │
  ├──────────────────┼──────────────────────────────────────────────────────────────────────┤
  │ 呼び元識別       │ 受け口配列（eArray[N]）の場合、添数により呼び元を識別できる          │
  ├──────────────────┼──────────────────────────────────────────────────────────────────────┤
  │ 合流受入可       │ 複数の呼び口が1つの受け口に結合できる（多対一）                      │
  ├──────────────────┼──────────────────────────────────────────────────────────────────────┤
  │ シグニチャ型付け │ 受け口は構文的整合性のみ保証。意味的整合性は各シグニチャの仕様が担う │
  └──────────────────┴──────────────────────────────────────────────────────────────────────┘

  結合の記述（組上げ記述）

  cell tCellType CellName {
      cCallPort = TargetCell.eEntryPort;   // 呼び口 → 受け口の結合
  };

  [through(plugin, arg)] 指定子でスルーセルを挿入：

  cCallPort = [through(RPCPlugin, "arg")] RemoteCell.eEntryPort;

  リージョンをまたぐ受け口

  受け口がリージョン内に置かれる場合、デフォルトでリージョン外からの結合は禁止されます。in_through
  指定でスルーセルを自動生成して許可できます：

  [in_through(RPCPlugin, "arg")] region rTrusted {
      cell tSecureCell sc { ... };
  };
  // 外部からの sc.ePort への結合が許可される（スルーセル挿入）

  RPC との関係

  受け口をまたぐ RPC 呼出しの分類（呼び口側の観点とも共通）：

  ┌────────────────────────────────────┬────────────────────────────────────────┐
  │              接続分類              │             受け口側の特性             │
  ├────────────────────────────────────┼────────────────────────────────────────┤
  │ トランスペアレント（直接）         │ 同一アドレス空間。通常の関数呼出し     │
  ├────────────────────────────────────┼────────────────────────────────────────┤
  │ トランスペアレント（アウトタスク） │ 同一 CPU・異なるタスク。メモリ共有あり │
  ├────────────────────────────────────┼────────────────────────────────────────┤
  │ オペイク（クローズドネットワーク） │ 静的接続。引数をコピー渡し             │
  ├────────────────────────────────────┼────────────────────────────────────────┤
  │ オペイク（オープンネットワーク）   │ 動的接続。認証・存在保証なし           │
  └────────────────────────────────────┴────────────────────────────────────────┘
