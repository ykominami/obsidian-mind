  「TECSコンポーネント図（TCD）」とは（tecs-0 wiki より）

  tecs-0 wiki での TCD の扱いは限定的です。TECS の4要素の一つとして概要レベルで言及されているのみです。

  TECS の4要素における位置づけ

  ┌───────────────────────────────┬──────────────────────────────────────────────────────┐
  │             要素              │                         内容                         │
  ├───────────────────────────────┼──────────────────────────────────────────────────────┤
  │ TECSコンポーネントモデル      │ セル・セルタイプ・シグニチャ・ポート・結合の概念定義 │
  ├───────────────────────────────┼──────────────────────────────────────────────────────┤
  │ TECSコンポーネント図（TCD）   │ コンポーネント構造の視覚的記法                       │
  ├───────────────────────────────┼──────────────────────────────────────────────────────┤
  │ コンポーネント記述言語（CDL） │ 組上げ記述と型定義のための形式言語                   │
  ├───────────────────────────────┼──────────────────────────────────────────────────────┤
  │ コンポーネント実装モデル      │ tecsgen が生成するC言語コードの構造規定              │
  └───────────────────────────────┴──────────────────────────────────────────────────────┘

  つまり TCD は「コンポーネント構造を視覚的に表現するための記法」です。CDL
  による形式的記述に対して、図として可視化するための表記体系に当たります。

  tecs-0 wiki はこの説明にとどまっており、具体的な記法（図のシンボル・接続の描き方等）は収録されていません。
