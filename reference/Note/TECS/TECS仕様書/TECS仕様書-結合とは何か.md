  TECS における「結合」とは（tecs-0 wiki より）

  **結合（join）とは、あるセルの呼び口（call port）を別のセルの受け口（entry
  port）**に接続する操作・その接続関係そのものです。TECSコンポーネントモデルの中核を成す概念の一つです。

  CDL での記述

  cell tCellType CellName {
    cCallPort = TargetCell.eEntryPort;   # 呼び口の結合
    attr = 42;                           # 属性の初期値
  };

  セル（インスタンス）の定義ブロック内で、自セルの呼び口名に対して 結合先セル名.受け口名
  を代入することで結合を宣言します。

  結合の基本ルール

  - 呼び口は必ず受け口と結合しなければならない（未結合はエラー）
  - 合流（多対一）は可能: 複数の呼び口が1つの受け口に結合できる
  - 分流（一対多）は不可: 1つの呼び口が複数の受け口に同時に結合することはできない
  - コールバック: 2つのセルが相互に呼び口・受け口を持ち接続し合う補助的な結合も存在する

  through 指定子によるスルーセル挿入

  [through(plugin, arg)] 指定子を使うと、結合の間にスルーセルを自動挿入できます（RPC チャネル生成等に利用）。

  リージョン間の結合

  セルが異なる region に属する場合、デフォルトではリージョン間の結合は禁止。許可するには region
  側で明示的に指定が必要です：
