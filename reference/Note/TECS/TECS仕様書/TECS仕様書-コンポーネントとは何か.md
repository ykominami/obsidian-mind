  コンポーネントとは何か（TECS における定義）

  TECS において「コンポーネント」は セル（cell）
  として実体化される、関数インタフェースを基本とするソフトウェア部品です。

  セル（コンポーネントインスタンス）

  セルは セルタイプ（celltype） のインスタンスであり、以下の4要素を持ちます：

  ┌──────────────────────┬───────────────────────────────────────────────────────────┐
  │         要素         │                           説明                            │
  ├──────────────────────┼───────────────────────────────────────────────────────────┤
  │ 受け口（entry port） │ 機能を提供する口。シグニチャに対応付けられる              │
  ├──────────────────────┼───────────────────────────────────────────────────────────┤
  │ 呼び口（call port）  │ 他のセルの機能を利用する口。シグニチャに対応付けられる    │
  ├──────────────────────┼───────────────────────────────────────────────────────────┤
  │ 属性                 │ ROM配置の読出し専用パラメータ（セルで初期値を上書き可能） │
  ├──────────────────────┼───────────────────────────────────────────────────────────┤
  │ 変数                 │ RAM配置の読み書き可能な内部状態                           │
  └──────────────────────┴───────────────────────────────────────────────────────────┘

  コンポーネントモデルの核心

  TECS コンポーネントには3つの重要な設計上の特徴があります：

  1. 静的な生成と結合:
  コンポーネントのインスタンス生成と結合はコンパイル時に確定する。実行時の動的生成・結合は基本的に対象外。
  2. 関数インタフェース基本: コンポーネント間通信は関数呼出しを基本とする。
  3. 形式的な型整合性: シグニチャ（signature） — 関数頭部の集合 — により、呼び口と受け口の型整合性が形式的に保証される。

  セルタイプ

  セルの型定義。セルの構造（口・属性・変数）と振舞い（セルタイプコード）を規定する。

  singleton   // システム中にセルが1つだけ存在
  idx_is_id   // 実行時識別に整数値を使用（有効性検査あり）
  active      // タスク等のCPU資源割付けを伴うアクティブセル

  呼び口・受け口の構文

  call  sSignature cPortName;    // 呼び口：機能を要求する
  entry sSignature ePortName;    // 受け口：機能を提供する

  呼び口は必ず受け口と結合しなければならない。複数の呼び口が1つの受け口に結合する「合流」（多対一）は可能だが、逆の「分
  流」は不可。

  命名規則

  ┌────────────┬──────────┬───────────────────┐
  │    要素    │ 接頭文字 │        例         │
  ├────────────┼──────────┼───────────────────┤
  │ シグニチャ │ s        │ sTask, sSemaphore │
  ├────────────┼──────────┼───────────────────┤
  │ セルタイプ │ t        │ tTask, tKernel    │
  ├────────────┼──────────┼───────────────────┤
  │ 呼び口     │ c        │ cTask, cKernel    │
  ├────────────┼──────────┼───────────────────┤
  │ 受け口     │ e        │ eTask, eKernel    │
  └────────────┴──────────┴───────────────────┘

  より大きな構造との関係

  複数のセルを組み合わせた 複合セルタイプ（composite celltype）
  を使うと、内部セルを隠蔽して新しいコンポーネントを構成できます。また リージョン（region）
  でセルの配置領域を指定し、リージョン間の結合を制限することでセキュリティ保護や RPC 挿入が可能です。
