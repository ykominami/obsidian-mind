  セルとは何か（tecs wiki 版）

  セル（cell） は TECS におけるコンポーネントのインスタンスです。セルタイプ（型定義）を元に生成され、システム中で実際に
  機能を実行する実体です。

  CDL でのセル定義

  セルは「組上げ記述（セル記述）」で生成します：

  cell tMyType MyCell {
      cCall = AnotherCell.eEntry;  // 呼び口に受け口を結合
      myAttr = 42;                 // 属性の初期値を上書き
  };

  セルが持つ要素：

  ┌──────────────────────┬──────┬────────────────────────────────────────────────────┐
  │         要素         │ 配置 │                        説明                        │
  ├──────────────────────┼──────┼────────────────────────────────────────────────────┤
  │ 受け口（Entry Port） │ —    │ 機能を提供する口。シグニチャで型規定               │
  ├──────────────────────┼──────┼────────────────────────────────────────────────────┤
  │ 呼び口（Call Port）  │ —    │ 他セルの機能を要求する口。必ず受け口と結合         │
  ├──────────────────────┼──────┼────────────────────────────────────────────────────┤
  │ 属性（Attribute）    │ ROM  │ 読出し専用パラメータ。セル記述で初期値を上書き可能 │
  ├──────────────────────┼──────┼────────────────────────────────────────────────────┤
  │ 変数（Variable）     │ RAM  │ 読み書き可能な内部状態                             │
  └──────────────────────┴──────┴────────────────────────────────────────────────────┘

  セル指定子

  ┌───────────┬────────────────────────────────────────────────────────┐
  │  指定子   │                          意味                          │
  ├───────────┼────────────────────────────────────────────────────────┤
  │ id(N)     │ セルタイプ内でのセル番号を明示指定                     │
  ├───────────┼────────────────────────────────────────────────────────┤
  │ prototype │ 前方参照用のプロトタイプ宣言。内部セルの相互参照に使用 │
  ├───────────┼────────────────────────────────────────────────────────┤
  │ allocator │ send/receive 引数がある受け口関数に必要                │
  └───────────┴────────────────────────────────────────────────────────┘

  セルタイプ指定子（型定義側）

  セルの振る舞いに影響するセルタイプ側の指定子：

  ┌───────────┬──────────────────────────────────────────────────┐
  │  指定子   │                       効果                       │
  ├───────────┼──────────────────────────────────────────────────┤
  │ singleton │ システム中にセルが1つだけ存在                    │
  ├───────────┼──────────────────────────────────────────────────┤
  │ active    │ タスク等、受け口関数が呼ばれなくても動作するセル │
  ├───────────┼──────────────────────────────────────────────────┤
  │ idx_is_id │ セルの実行時識別に整数値を使用                   │
  └───────────┴──────────────────────────────────────────────────┘

  C言語実装でのセルの表現

  tecsgen がセルから生成する C 言語の構造：

  - CELLIDX — セルを識別するインデックス。受け口関数の第1引数
  - CELLCB（Cell Control Block） — セルの CB ポインタ。属性・変数へのアクセスに使用
  - VALID_IDX(idx) — IDX の正当性チェック
  - GET_CELLCB(idx) — CB ポインタ取得

  非シングルトンセルの受け口関数の骨格：

  ER eEntry_func(CELLIDX idx, int32_t inval, int32_t *outval) {
      CELLCB *p_cellcb;
      if (VALID_IDX(idx)) {
          p_cellcb = GET_CELLCB(idx);
      } else {
          return E_ID;
      }
      *outval = inval - ATTR_myAttr;   // ROM 属性にアクセス
      return E_OK;
  }

  シングルトンセルでは CELLIDX idx 引数と CB 取得が不要になります。
