  コンポーネントとは何か（tecs wiki 版）

  TECS において「コンポーネント」は セル（cell） と呼ばれるインスタンスとして実体化される、機能の実装単位です。

  コンポーネントの構成要素

  ┌────────────────────────┬────────────────────────────────────────────────────────┐
  │          要素          │                          説明                          │
  ├────────────────────────┼────────────────────────────────────────────────────────┤
  │ コンポーネント（セル） │ 機能の実装単位。セルタイプのインスタンス               │
  ├────────────────────────┼────────────────────────────────────────────────────────┤
  │ セルタイプ             │ コンポーネントの型定義。属性・変数・ポートを持つ       │
  ├────────────────────────┼────────────────────────────────────────────────────────┤
  │ 呼び口（Call Port）    │ 機能を要求するポート                                   │
  ├────────────────────────┼────────────────────────────────────────────────────────┤
  │ 受け口（Entry Port）   │ 機能を提供するポート                                   │
  ├────────────────────────┼────────────────────────────────────────────────────────┤
  │ シグニチャ             │ 呼び口・受け口のインターフェース定義（関数ヘッダ集合） │
  └────────────────────────┴────────────────────────────────────────────────────────┘

  CDL でのコンポーネント記述

  コンポーネント構造は CDL（Component Description Language） で記述します。CDL の標準的な記述順：

  1. 前置部（Preamble） — C ヘッダのインポート、typedef、struct、定数定義
  2. シグニチャ記述 — インターフェース定義（関数ヘッダ集合）
  3. セルタイプ記述 — コンポーネント型の定義
  4. 複合セルタイプ記述 — 複数セルを内包するコンポーネント型の定義
  5. 組上げ記述（セル記述） — インスタンス生成と結合
  6. ネームスペース / リージョン — 名前衝突防止・構造化

  コンポーネントを扱うツールチェーン
