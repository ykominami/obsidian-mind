  シグニチャとは何か

  シグニチャ（signature）は、関数ヘッダの集合であり、TECS コンポーネント間のインターフェース定義です。 呼び口（Call
  Port）と受け口（Receptacle）の両方に対応します。

  CDL の記述構造において、シグニチャはシグニチャ記述として前置部（import_C / typedef / struct
  等）の直後に書くのが標準的な順序です。

  役割

  コンポーネント間のやり取りを定義するインターフェース仕様。どのような関数を（どのパラメータで）呼び出せるかを宣言します
  。

  基本的な指定子

  ┌──────────────────────────────────┬─────────────────────────────────────────────────────┐
  │              指定子              │                        意味                         │
  ├──────────────────────────────────┼─────────────────────────────────────────────────────┤
  │ callback                         │ コールバック用（受け口→呼び口の逆結合を可能にする） │
  ├──────────────────────────────────┼─────────────────────────────────────────────────────┤
  │ context("task"|"non-task"|"any") │ 実行コンテキスト（設計ドキュメント用）              │
  ├──────────────────────────────────┼─────────────────────────────────────────────────────┤
  │ deviate                          │ 標準規則に違反する引数を持つことを示す              │
  └──────────────────────────────────┴─────────────────────────────────────────────────────┘

  パラメータ方向指定子

  シグニチャの各関数引数には方向を指定します。

  ┌─────────┬───────────────┬────────────┐
  │ 指定子  │     方向      │ メモリ解放 │
  ├─────────┼───────────────┼────────────┤
  │ in      │ 呼び元→呼び先 │ なし       │
  ├─────────┼───────────────┼────────────┤
  │ out     │ 呼び元←呼び先 │ なし       │
  ├─────────┼───────────────┼────────────┤
  │ inout   │ 双方向        │ なし       │
  ├─────────┼───────────────┼────────────┤
  │ send    │ 呼び元→呼び先 │ あり       │
  ├─────────┼───────────────┼────────────┤
  │ receive │ 呼び元←呼び先 │ あり       │
  └─────────┴───────────────┴────────────┘

  send / receive はメモリ解放を伴う点が通常の in / out と異なります。

  ポインタ修飾指定子

  ┌─────────────────┬────────────────────────────────────┐
  │     指定子      │                意味                │
  ├─────────────────┼────────────────────────────────────┤
  │ size_is(param)  │ 配列を指す。サイズは param で指定  │
  ├─────────────────┼────────────────────────────────────┤
  │ count_is(param) │ 有効要素数（size_is と組み合わせ） │
  ├─────────────────┼────────────────────────────────────┤
  │ string          │ NULL 終端文字列                    │
  ├─────────────────┼────────────────────────────────────┤
  │ nullable        │ NULL ポインタ渡しを許可            │
  └─────────────────┴────────────────────────────────────┘

  命名規則

  シグニチャ名には慣習的な接頭文字があります。

  ┌──────────────────────┬──────────┬───────────────┐
  │     コンテキスト     │ 接頭文字 │      例       │
  ├──────────────────────┼──────────┼───────────────┤
  │ タスクコンテキスト   │ s        │ sMotor, sLamp │
  ├──────────────────────┼──────────┼───────────────┤
  │ 非タスクコンテキスト │ si       │ siTask        │
  ├──────────────────────┼──────────┼───────────────┤
  │ カーネル外           │ sn       │ snInit        │
  └──────────────────────┴──────────┴───────────────┘

  グローバル名（生成規則）

  tecsgen が生成する C 言語識別子のパターン:

  (NS名)_(シグニチャ名)

  生成ファイルは SIGNATURE_tecsgen.h（シグニチャ宣言）。

  実例（ASP3）

  ASP3 カーネルオブジェクトでの使用例：sTask（13関数）、sSemaphore、sEventflag、sDataqueue
  など、各カーネルオブジェクトごとにシグニチャが定義されています。
