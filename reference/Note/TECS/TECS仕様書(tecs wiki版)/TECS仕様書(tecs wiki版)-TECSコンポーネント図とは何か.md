  「TECSコンポーネント図（TCD）」とは（tecs wiki より）

  **TECS コンポーネント図（TCD: TECS Component Diagram）**は、TECS システムの構造を表す図です。CDL
  の組上げ記述（属性値を除く）と1対1に対応するため、図と CDL を相互に変換できます。

  描画原則

  ▎ 上位レイヤ（アプリケーション）を上または左、下位レイヤ（デバイスドライバ・OSカーネル）を下または右に配置する。

  基本要素と記法

  ┌────────────────┬─────────────────────────────────────────────────┬──────────────────────────────────┐
  │      要素      │                    TCD 表記                     │             CDL 記述             │
  ├────────────────┼─────────────────────────────────────────────────┼──────────────────────────────────┤
  │ セル           │ 矩形（内部にセルタイプ名とインスタンス名）      │ cell tCelltype Name { ... };     │
  ├────────────────┼─────────────────────────────────────────────────┼──────────────────────────────────┤
  │ 受け口         │ 矩形辺に内向きに付く塗りつぶし三角形 + ポート名 │ entry ポート宣言                 │
  ├────────────────┼─────────────────────────────────────────────────┼──────────────────────────────────┤
  │ 呼び口         │ 矩形から伸びる線分 + ポート名                   │ call ポート宣言                  │
  ├────────────────┼─────────────────────────────────────────────────┼──────────────────────────────────┤
  │ 接続線         │ 呼び口 → 受け口。線上にシグニチャ名             │ cPortName = OtherCell.ePortName; │
  ├────────────────┼─────────────────────────────────────────────────┼──────────────────────────────────┤
  │ アクティブセル │ 矩形輪郭を二重線                                │ [active] celltype                │
  ├────────────────┼─────────────────────────────────────────────────┼──────────────────────────────────┤
  │ RPC 接続       │ 接続線を二重線                                  │ RPC プラグインによる生成         │
  └────────────────┴─────────────────────────────────────────────────┴──────────────────────────────────┘

  特殊な接続形態

  ┌────────────────────────┬─────────────────────────────────────────────────────────────────────────────┐
  │          形態          │                                    説明                                     │
  ├────────────────────────┼─────────────────────────────────────────────────────────────────────────────┤
  │ 合流                   │ 複数の呼び口が1つの受け口に接続（逆の分流は不可）。3本の線が集まる形で表記  │
  ├────────────────────────┼─────────────────────────────────────────────────────────────────────────────┤
  │ 呼び口配列・受け口配列 │ 配列インデックスをポート名に付加して表記                                    │
  ├────────────────────────┼─────────────────────────────────────────────────────────────────────────────┤
  │ コールバック           │ callback 指定子のシグニチャを使い、受け口から呼び口への逆方向呼び出しを表現 │
  └────────────────────────┴─────────────────────────────────────────────────────────────────────────────┘

  高度な表記

  - アロケータ: ▼ シンボルを接続線近傍に配置し、アロケータセルの受け口に繋げる
  - 多段リレーモデル: メモリ領域が受け口で確保・受け取られ、呼び口への send 引数として渡されるモデル。tecsgen
  がアロケータ呼び口を内部的に自動生成

  複合セル（Composite）の表記

  複合セルには特別なマーキングなし（通常セルと同じ見た目）。内部構造は通常コンポーネント図には記載しません。
