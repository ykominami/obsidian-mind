  TECSコンポーネント図（TCD）の書き方（tecs wiki より）

  TCD は CDL の組上げ記述（属性値を除く）と1対1に対応するため、CDL が読めれば図に変換でき、逆も然りです。

  ---
  Step 1 — 配置の原則を決める

  ▎ 上位レイヤ（アプリケーション）を上または左、下位レイヤ（デバイスドライバ・OS カーネル）を下または右に配置する。

  ---
  Step 2 — 基本要素を描く

  ┌────────┬────────────────────────────────────────────────┬──────────────────────────────┐
  │  要素  │                     描き方                     │           対応 CDL           │
  ├────────┼────────────────────────────────────────────────┼──────────────────────────────┤
  │ セル   │ 矩形。内部にセルタイプ名とインスタンス名を記載 │ cell tCelltype Name { ... }; │
  ├────────┼────────────────────────────────────────────────┼──────────────────────────────┤
  │ 受け口 │ 矩形辺に内向きの塗りつぶし三角形 + ポート名    │ entry sSignature eXxx;       │
  ├────────┼────────────────────────────────────────────────┼──────────────────────────────┤
  │ 呼び口 │ 矩形から伸びる線分 + ポート名                  │ call sSignature cXxx;        │
  ├────────┼────────────────────────────────────────────────┼──────────────────────────────┤
  │ 接続線 │ 呼び口 → 受け口。線上にシグニチャ名を記載      │ cXxx = OtherCell.eXxx;       │
  └────────┴────────────────────────────────────────────────┴──────────────────────────────┘

  ---
  Step 3 — セル種別に応じた修飾を加える

  ┌────────────────┬──────────────────────────────────┬────────────────────┐
  │    セル種別    │              描き方              │        CDL         │
  ├────────────────┼──────────────────────────────────┼────────────────────┤
  │ 通常セル       │ 単線矩形                         │ 通常の celltype    │
  ├────────────────┼──────────────────────────────────┼────────────────────┤
  │ アクティブセル │ 二重線矩形                       │ [active] celltype  │
  ├────────────────┼──────────────────────────────────┼────────────────────┤
  │ 複合セル       │ 単線矩形（特別なマーキングなし） │ composite celltype │
  └────────────────┴──────────────────────────────────┴────────────────────┘

  ---
  Step 4 — 特殊な接続形態を正しく描く

  合流（多対一）

  複数の呼び口を1つの受け口に接続できます。3本の線が1点に集まる形で描きます。

  ▎ ※ 4本の線が交わっている場合は「別々の線が交差しているだけ」（合流ではない）

  呼び口配列・受け口配列

  配列インデックスをポート名に付加して表記します（例:
  cCall[0]、eEntry[1]）。呼び口配列と受け口配列のインデックスを対応させると、複数の呼び元にわたるコールバック機能を実現できます。

  コールバック

  callback 指定子を持つシグニチャを使った接続。受け口から呼び口へ逆方向の呼び出しを実現します。

  ---
  Step 5 — 高度な要素（必要に応じて）

  RPC（遠隔手続き呼び出し）

  接続線を二重線で描きます。実際には RPC チャネルコンポーネントに置き換えられます。

  アロケータ

  ▼ シンボルを接続線近傍に配置し、アロケータセルの受け口に繋げます。
  - send / receive パラメータに対して tecsgen がアロケータ呼び口を自動生成します
  - CDL の組上げ記述では受け口単位でアロケータを指定します

  多段リレーモデル

  メモリ領域が受け口で確保・受け取られ、呼び口への send 引数として渡されるモデル。
  - 最右辺の受け口でアロケータを指定します
  - tecsgen が送り元・受け元それぞれのアロケータ呼び口を内部的に生成します
