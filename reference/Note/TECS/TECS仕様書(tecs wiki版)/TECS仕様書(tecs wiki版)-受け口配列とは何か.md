  「受け口配列」とは（tecs-0 wiki より）

  **受け口配列（entry port
  array）**とは、1つのセルタイプが複数の呼び元（呼び口）を同時に受け付けられるよう、受け口をサイズ付きの配列として宣言したものです。

  CDL での宣言構文

  entry sSignature eArray[N];    # 受け口配列（サイズ N を明示）

  通常の受け口（entry sSignature ePortName;）との違いは、インデックス番号によって複数の呼び元を区別できる点です。

  受け口配列の最大の特徴：呼び元の識別

  ▎ 受け口：機能を提供する口。呼び元を識別できる（受け口配列の場合）。

  受け口配列を使うと、セルタイプコードの中で「どのインデックス（どの呼び元）から呼ばれたか」を判定できます。これにより、同一のセルタイ
  プで複数の呼び元に対して異なる処理を行うことが可能になります。

  呼び口配列との対比

  call  sSignature cArray[];     # 呼び口配列 — 呼び出す側、サイズ省略可
  entry sSignature eArray[N];    # 受け口配列 — 提供する側、サイズ N を明示

  │    項目    │          呼び口配列          │                  受け口配列                  │
  ├────────────┼──────────────────────────────┼──────────────────────────────────────────────┤
  │ 役割       │ 複数の受け口を呼び出す       │ 複数の呼び元を受け付ける                     │

  ├────────────┼──────────────────────────────┼──────────────────────────────────────────────┤
  │ 識別情報   │ —                            │ セルタイプコードで呼び元インデックスを参照可 │
  └────────────┴──────────────────────────────┴──────────────────────────────────────────────┘

  実装モデルへの影響

  受け口配列かどうかは tecsgen の最適化判定に影響します：

  - 受け口が配列でない場合 → 受け口ディスクリプタ不要の最適化が適用可能（受け口ディスクリプタを IDX に置換）
  - 受け口が配列の場合 → 受け口ディスクリプタが必要となるため、その最適化は適用不可

  INIB（ROM配置ブロック）には「呼び口結合先受け口ディスクリプタへのポインタ」が格納されますが、受け口配列の場合はこのディスクリプタが必
  要になります。
