# LLM Wiki v2 — メモリ設計パターン

出典: [LLM Wiki v2 — エージェントメモリ構築の教訓を取り入れたKarpathyのLLM Wikiパターンの拡張 · GitHub](https://gist.github.com/rohitg00/2067ab416f7bbe447c1977edaaa681e2)

## メモリライフサイクル

### 信頼度スコアリング

- 裏付ける情報源の数・最後に確認された時期・矛盾する情報の有無などで算出
- 信頼度は時間とともに低下し、強化されると高まる

### 置き換え

- 新しい情報が既存の主張と矛盾・更新する場合、古い主張は注釈付きで残さず明示的に置き換える
- リンク・タイムスタンプ・古いバージョンは保存するが、古いバージョンは「古い」とマークする

### 忘却

- 決して忘れない Wiki はノイズだらけになる
- 保持曲線を実装 — 削除ではなく優先順位を下げる
- 保持は時間とともに指数関数的に減少、確認（アクセス・新しい情報源）でリセット
- アーキテクチャ上の決定はゆっくり減衰、一時的なバグは急速に減衰

### 統合階層

| 階層 | 説明 |
|---|---|
| ワーキングメモリ | 最近観察された情報（未処理） |
| エピソード記憶 | 観察データから圧縮されたセッション要約 |
| 意味記憶 | エピソードから統合された、セッションをまたいだ事実 |
| 手続き的記憶 | 反復的な意味から抽出されたワークフローとパターン |

各階層は下の階層より圧縮・確実・長期的。証拠が蓄積されるにつれて上位階層へ昇格。

## 知識グラフ

- ページの上に重ねられた型付き知識グラフ
- エンティティ抽出: 人物・プロジェクト・ライブラリ・概念・ファイル・決定事項など
- 各エンティティに型・属性・他エンティティとの関係を割り当て
- 関係性には型がある: 「使用する」「依存する」「矛盾する」「原因となる」「修正する」「置き換える」

## 実際に拡張可能な検索

### オリジナル版の限界

- 単一の `index.md` に依存 — 100〜200ページで機能するが、それ以上は長くなりすぎる

### ハイブリッド検索

- BM25（語幹と類義語展開によるキーワードマッチング）
- ベクトル検索（埋め込みによる意味的類似性）
- グラフ操作（エンティティ認識型リレーションシップ検索）
- 結果を相互ランクで統合
- `index.md` は人間が読む用。検索では頼らない
