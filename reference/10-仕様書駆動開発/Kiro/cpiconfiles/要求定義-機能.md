# 要件定義書

## はじめに

本ドキュメントは、アイコンファイル管理システム（icon-file-group-manager）の要件を定義する。
本システムは、指定ディレクトリ配下に存在するアイコンファイルをグループ単位で管理し、スナップショットによる履歴管理、および指定条件に合致するアイコンファイルのコピー機能を提供する。

## 用語集

- **Icon_File**: アイコンとして使用される画像ファイル（PNG、SVG 等）。ファイルパスと SHA256 ハッシュ値の組で一意に識別される。
- **Icon_File_Group**: 同じ種類の複数サイズのアイコンファイル群をまとめた論理的な単位。
- **Snapshot**: 指定ディレクトリ配下に存在するアイコンファイルグループ群の状態を記録したもの。「指定ディレクトリ」と「スナップショット取得回数（連番）」で識別される。
- **Snapshot_History**: 複数のスナップショットから生成される履歴オブジェクト。各アイコンファイルグループの有効・無効状態を管理する。
- **SHA256**: ファイル内容の整合性検証に使用する 256 ビットのハッシュ値。
- **Source_Directory**: スナップショット取得対象となるアイコンファイルが格納されたディレクトリ。
- **Destination_Directory**: アイコンファイルのコピー先となるディレクトリ。
- **Copy_Mapping**: コピー元アイコンファイルグループとコピー先アイコンファイルグループの対応関係。
- **Manager**: 本システム全体を指す名称。

## 要件

### 要件 1: アイコンファイルの識別

**ユーザーストーリー:** 開発者として、アイコンファイルをファイルパスと内容の両面から一意に識別したい。それにより、ファイル名が同じでも内容が異なるファイルを区別できる。

#### 受け入れ基準

1. THE Manager SHALL アイコンファイルをファイルパスと SHA256 ハッシュ値の組として識別する。
2. WHEN アイコンファイルの内容が変更された場合、THE Manager SHALL 変更前後のファイルを異なるアイコンファイルとして扱う。
3. WHEN 同一パスに同一 SHA256 ハッシュ値を持つファイルが存在する場合、THE Manager SHALL それらを同一のアイコンファイルとして扱う。
4. IF ファイルパスが存在しない場合、THEN THE Manager SHALL エラーを返す。
5. IF ファイルの読み取り権限がない場合、THEN THE Manager SHALL 権限エラーを返す。

---

### 要件 2: アイコンファイルグループ管理

**ユーザーストーリー:** 開発者として、同じ種類の複数サイズのアイコンファイルをグループとして管理したい。それにより、サイズ違いのアイコンを一括して扱える。

#### 受け入れ基準

1. THE Manager SHALL 同じ種類の複数サイズのアイコンファイルを 1 つの Icon_File_Group として管理する。
2. THE Manager SHALL 1 つの Icon_File_Group に対して 1 つ以上の Icon_File を関連付ける。
3. WHEN Source_Directory をスキャンする場合、THE Manager SHALL 同一ディレクトリ内に存在しかつファイル名のプレフィックスが同一であるファイルを同一の Icon_File_Group として自動的に識別する。
4. WHEN Icon_File_Group を構成する場合、THE Manager SHALL グループ名として「ディレクトリパス」と「共通プレフィックス」の組を使用する。
5. THE Manager SHALL Icon_File_Group に含まれる各 Icon_File のサイズ情報（幅・高さのピクセル値）を保持する。
6. WHEN Icon_File のサイズ情報を取得する場合、THE Manager SHALL 画像ファイルを解析してピクセル単位の幅・高さを取得する。
7. IF Icon_File_Group に同一サイズの Icon_File が複数含まれる場合、THEN THE Manager SHALL エラーを返す。

---

### 要件 3: 指定ディレクトリのスキャン

**ユーザーストーリー:** 開発者として、指定ディレクトリ配下のアイコンファイルを自動的に検出したい。それにより、手動でファイルを登録する手間を省ける。

#### 受け入れ基準

1. WHEN Source_Directory が指定された場合、THE Manager SHALL そのディレクトリ配下を再帰的にスキャンし、アイコンファイルを検出する。
2. WHEN アイコンファイルを検出した場合、THE Manager SHALL 各ファイルの SHA256 ハッシュ値を計算する。
3. WHEN アイコンファイルを検出した場合、THE Manager SHALL 画像ファイルを解析して各ファイルのピクセル単位の幅・高さを取得する。
4. THE Manager SHALL スキャン結果として、検出した全 Icon_File のファイルパス、SHA256 ハッシュ値、およびサイズ情報（幅・高さ）の一覧を返す。
5. IF Source_Directory が存在しない場合、THEN THE Manager SHALL エラーを返す。
6. IF Source_Directory が空である場合、THEN THE Manager SHALL 空の一覧を返す。
7. IF 画像ファイルの解析に失敗した場合、THEN THE Manager SHALL エラーを返す。

---

### 要件 4: スナップショット管理

**ユーザーストーリー:** 開発者として、特定時点のアイコンファイルグループ群の状態をスナップショットとして記録したい。それにより、過去の状態を参照・比較できる。

#### 受け入れ基準

1. WHEN スナップショットを取得する場合、THE Manager SHALL Source_Directory と取得時点の Icon_File_Group 群の状態を記録する。
2. THE Manager SHALL Snapshot を「Source_Directory のパス」と「スナップショット取得回数（1 から始まる連番）」の組で識別する。
3. WHEN 同一 Source_Directory に対して複数回スナップショットを取得した場合、THE Manager SHALL 取得回数を 1 ずつ増加させる。
4. THE Manager SHALL 各 Snapshot に含まれる全 Icon_File_Group の一覧を保持する。
5. WHEN Snapshot を取得する場合、THE Manager SHALL 取得日時を記録する。
6. IF 指定した Snapshot が存在しない場合、THEN THE Manager SHALL エラーを返す。

---

### 要件 5: スナップショットヒストリー管理

**ユーザーストーリー:** 開発者として、複数のスナップショットから履歴を生成し、各アイコンファイルグループの有効・無効を管理したい。それにより、変更履歴を追跡し、特定バージョンのアイコンを有効化・無効化できる。

#### 受け入れ基準

1. WHEN 1 つ以上の Snapshot が存在する場合、THE Manager SHALL それらから 1 つの Snapshot_History を生成する。
2. THE Manager SHALL Snapshot_History において各 Icon_File_Group に対して有効または無効の状態を管理する。
3. WHEN Snapshot_History を生成する場合、THE Manager SHALL 全 Snapshot に含まれる Icon_File_Group を網羅する。
4. WHEN Icon_File_Group の有効・無効状態を変更する場合、THE Manager SHALL 変更後の状態を Snapshot_History に反映する。
5. THE Manager SHALL Snapshot_History に含まれる有効な Icon_File_Group の一覧を返す機能を提供する。
6. THE Manager SHALL Snapshot_History に含まれる無効な Icon_File_Group の一覧を返す機能を提供する。
7. IF Snapshot が 1 つも存在しない場合、THEN THE Manager SHALL エラーを返す。

---

### 要件 6: アイコンファイルのコピー機能

**ユーザーストーリー:** 開発者として、指定サイズ群を含むアイコンファイルグループを選択し、コピー先ディレクトリにコピーしたい。それにより、必要なサイズのアイコンを別の場所に配置できる。

#### 受け入れ基準

1. WHEN コピー操作を実行する場合、THE Manager SHALL 指定されたサイズ群を全て含む Icon_File_Group を Snapshot_History の有効なグループの中から候補として列挙する。
2. WHEN コピー操作を実行する場合、THE Manager SHALL 候補 Icon_File_Group が複数存在するとき、Copy_Mapping に既に記録済みの Icon_File_Group を候補から除外する。
3. WHEN 除外後の候補が 1 つになった場合、THE Manager SHALL その Icon_File_Group をコピー対象として自動選択する。
4. WHEN 除外後の候補が複数残る場合、THE Manager SHALL 候補の一覧をユーザーに提示し、ユーザーが 1 つを選択できるようにする。
5. WHEN ユーザーが候補から 1 つの Icon_File_Group を選択した場合、THE Manager SHALL その Icon_File_Group をコピー対象として使用する。
6. WHEN コピー対象の Icon_File_Group が決定した場合、THE Manager SHALL その Icon_File_Group に含まれる全 Icon_File を Destination_Directory にコピーする。
7. THE Manager SHALL コピー元 Icon_File_Group とコピー先 Icon_File_Group の対応を Copy_Mapping として記録する。
8. WHEN コピーが完了した場合、THE Manager SHALL コピー先の各 Icon_File の SHA256 ハッシュ値を検証し、コピー元と一致することを確認する。
9. IF 指定されたサイズ群を全て含む Icon_File_Group が存在しない場合、THEN THE Manager SHALL エラーを返す。
10. IF Copy_Mapping 記録済みを除外した結果、候補が 0 になった場合、THEN THE Manager SHALL エラーを返す。
11. IF Destination_Directory が存在しない場合、THEN THE Manager SHALL Destination_Directory を作成してからコピーを実行する。
12. IF コピー後の SHA256 ハッシュ値がコピー元と一致しない場合、THEN THE Manager SHALL エラーを返す。

---

### 要件 7: データの永続化

**ユーザーストーリー:** 開発者として、管理データをファイルに保存・読み込みしたい。それにより、プロセスを再起動しても管理状態を維持できる。

#### 受け入れ基準

1. THE Manager SHALL Icon_File_Group、Snapshot、Snapshot_History、Copy_Mapping のデータを永続化する。
2. WHEN データを保存する場合、THE Manager SHALL 指定されたストレージパスにデータを書き込む。
3. WHEN データを読み込む場合、THE Manager SHALL 指定されたストレージパスからデータを読み込む。
4. IF 保存先のファイルが破損している場合、THEN THE Manager SHALL エラーを返す。
5. THE Manager SHALL 保存・読み込みにおいてデータの整合性を保証する（保存したデータを読み込むと同一のデータが復元される）。
