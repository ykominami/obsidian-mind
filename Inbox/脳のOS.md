# Project
今週のタスク
進行中の企画
未処理のアイデア
（例）
- クライアントへの納品物
- 執筆中の記事
- 進行中のプロジェクトのデータ
**ルール**  
ここには「今動いているもの」だけを置く。終わったプロジェクトは別の場所（完了フォルダなど）へ移す。常にキッチンを清潔に保つイメージです。
# Knowledge
学びのメモ
個人的な哲学
フレームワーク集
(例)
- 思考の断片メモ（`thought.txt`）
    
- 自分のプロフィール・自己紹介文
    
- 仕事における判断基準（スタイルガイド）
    
- 戦略や今後のビジョン
**なぜこれが重要か？  
**AIがこのフォルダを読むことで、「あなたらしい」アウトプットを出せるようになるからです。ここがないと、AIは常に「一般的な回答」しかできません。**Knowledgeフォルダは、AIに「あなたの人格」をインストールするための場所**なのです。
# Reference
AIプロンプト集
記事ネタ帳
外部レポート
(例)
- **AIプロンプト集（画像生成、記事執筆、リサーチ用など）**
    
- **記事のネタ帳・テーマリスト**
    
- **参考になった外部資料やレポート**
    
- **業界のトレンドまとめ**
ルール
「いつか使うかも」ではなく、「これは確実に再利用する」と思えるものだけを厳選して保管する。
# Asset
画像素材
デザインテンプレート
ロゴ・アイコン

(例)
- **背景画像、人物写真**
    
- **アイコン、イラスト素材**
    
- **ブランドロゴやフォントデータ**
**ルール**  
素材はカテゴリごと（人物、背景、アイコンなど）にサブフォルダで分ける。「あの画像どこだっけ？」を二度と言わないために。
